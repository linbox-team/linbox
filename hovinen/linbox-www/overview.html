What is Project Linbox?

Project Linbox is a C++ template library for high-performance exact sparse computational linear algebra. What does all this mean?

  • Computational linear algebra. Linbox can solve linear systems, find least-norm, least-squares solutions to singular and inconsistent systems, and compute several matrix invariants, such as minimal and characteristic polynomials, rank, and determinant.
  • Sparse. Linbox is designed primarily to work with sparse matrices, which are defined in this context as those matrices where the computational cost of application of an m by n matrix to a vector is significantly less than O(mn), as would be the case for a dense matrix.
  • Exact. All of Linbox's computational results are exact, unlike the case with numerical systems like BLAS or Matlab. Linbox can be used to provide the computational infrastructure needed for rigorous theorem-proving in addition to number-crunching.
  • High-performance. Linbox implements iterative system-solving methods such as Wiedemann and Lanczos to operate on very large, sparse linear systems. This avoids the potential fill-in associated with elimination-based methods and keeps memory use relatively constant through the computation. These methods allow Linbox to be used to solve systems with hundreds of thousands of equations and hundreds of thousands of variables.
  • C++ template library. C++ templates are used to provide both high performance and genericity. In particular, Linbox algorithms are generic with respect to the field over which they operate.

Project goals

Linbox aims to provide world-class implementations of the most advanced algorithms to date for exact linear algebra.

Functionality:

  • System solutions:
    • Nonsingular systems
    • Random solutions for consistent singular systems
    • Nullspace basis and random sample
    • Least-norm, least-squares solutions
    • Certificate of inconsistency
    • Over a finite field
    • Integer solutions (i.e., Diophantine systems)
    • Rational solutions, using rational reconstruction
  • Linear operator invariants:
    • Determinant
    • Trace
    • Rank [1]
    • Minimal polynomial
    • Characteristic polynomial
    • Eigenvalues/eigenvectors
    • Similarity testing
  • Matrix canonical forms:
    • Smith form
    • Rational Jordan form
    • Frobenius form
[1] These methods are probabilistic for a finite field, though for rank there exists a certification when the field is of characteristic zero.
